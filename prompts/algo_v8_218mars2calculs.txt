## DISTINCTION CRITIQUE : PILIER QUESTION vs PILIER RÉPONSE CŒUR

**PILIER DE LA QUESTION** (`pilier_question`) :
- C'est ce qu'ON DEMANDE au candidat
- Le STIMULUS qu'on lui donne pour le tester
- Exemple : "P5" → on lui demande d'exécuter
- C'est notre INTENTION de test

**PILIER RÉPONSE CŒUR** (`pilier_reponse_coeur`) :
- C'est ce que le candidat FAIT RÉELLEMENT dans sa réponse
- Son PROCESSUS COGNITIF INCONSCIENT activé
- Exemple : "P4" → il conçoit au lieu d'exécuter
- C'est SA RÉALITÉ COGNITIVE

⚠️ **RÈGLE ABSOLUE DU SCORING :**
> On voulait le mettre dans un pilier (pilier_question), mais il a choisi son pilier pour répondre (pilier_reponse_coeur). C'est ce qu'il fait INCONSCIEMMENT.
> 
> **C'est CETTE BASE (pilier_reponse_coeur) qui nous intéresse UNIQUEMENT pour le scoring, car c'est la RÉALITÉ COGNITIVE du candidat.**

## DISTRIBUTION THÉORIQUE vs DISTRIBUTION RÉELLE

**CE QU'ON FAIT (distribution théorique) :**
- 25 questions réparties sur 5 piliers : **5-5-5-5-5**
- Chaque pilier reçoit 5 questions de test
- C'est notre STIMULUS standardisé

**CE QUE LE CANDIDAT FAIT (distribution réelle) :**
- Le candidat RÉPOND avec SA distribution cognitive réelle
- Exemple Cécile : **P1=7, P2=2, P3=8, P4=5, P5=3**
- Cette distribution réelle = SA SIGNATURE COGNITIVE

**Exemple concret :**
- Question Q21 : pilier_question = "P5" (on demande exécution)
- Réponse candidat : pilier_reponse_coeur = "P4" (il conçoit au lieu d'exécuter)
- **Pour le scoring : Q21 est comptée dans P4, PAS dans P5**

→ C'est comme ça qu'on découvre que le candidat a **3 questions en P5** (écart -2) et **5 questions en P4** (écart 0).

## SYSTÈME 1 : SCORE BARÈME (contrôle qualité)

**OBJECTIF :** Vérifier que les dimensions reflètent bien le niveau observé.
**USAGE :** Système de CONTRÔLE. On le calcule MAIS on ne s'en sert PAS pour le profil final.

### **FORMULE COMPLÈTE :**

```javascript
// ÉTAPE 1 : Score dimensions
score_dimensions = (simples × 0.30) + (details × 0.20) + (soph × 0.90)

Où :
- 1 simple = 0.30
- 1 détail = 0.20 (détails liés aux simples uniquement)
- 1 sophistiquée = 0.90 (= 1 simple + 3 détails = 0.30 + 0.60)

// ÉTAPE 2 : Plafond niveau 6 DÉTECTEUR
score_dimensions_plafonné = MIN(score_dimensions, 2.75)

Note : 2.75 = milieu de la plage du niveau 6 (2.50-2.99)
Le score base (dimensions seules) ne peut JAMAIS dépasser le milieu du niveau 6.

// ÉTAPE 3 : Bonus excellences
bonus_excellences = 
  mapExcellence(anticipation) +
  mapExcellence(decentration_niveau) +
  mapExcellence(metacognition_niveau) +
  mapExcellence(vue_systemique)

Mapping excellences :
- nulle → 0.00
- faible → 0.08
- moyen → 0.15
- élevé → 0.25
Maximum : 4 × 0.25 = 1.00

// ÉTAPE 4 : Score barème final
score_bareme = score_dimensions_plafonné + bonus_excellences
score_bareme = MIN(score_bareme, 4.00)  // Plafond absolu
```

### **EXEMPLE CALCUL Q11 (Cécile) :**

Données :
- simples = 4, details = 7, soph = 4
- excellences : anticip=faible(0.08), decent_cogn=nulle(0), meta=élevé(0.25), vue_syst=élevé(0.25)

Calcul :
```
score_dimensions = (4 × 0.30) + (7 × 0.20) + (4 × 0.90)
                 = 1.20 + 1.40 + 3.60
                 = 6.20

score_plafonné = MIN(6.20, 2.75) = 2.75

bonus_excellences = 0.08 + 0 + 0.25 + 0.25 = 0.58

score_bareme = 2.75 + 0.58 = 3.33
```

## SYSTÈME 2 : SCORE CONTENU avec ajustement dimensions ← CELUI QUI COMPTE

**OBJECTIF :** Refléter la RÉALITÉ COGNITIVE observée par l'Agent 1 avec ajustement fin par les dimensions.
**USAGE :** C'est CE score qui est utilisé pour le profil final du candidat.

### **FORMULE COMPLÈTE :**

```javascript
// ÉTAPE 1 : Score niveau (milieu de plage selon amplitude observée)
niveau_amplitude_max = valeur fournie par Agent 1 (1-9)
score_niveau = mapping_milieu_plage[niveau_amplitude_max]

Mapping niveau → score (milieu de plage) :
1 EXÉCUTEUR      (0.00-0.49)  → 0.25
2 SYSTÉMATIQUE   (0.50-0.99)  → 0.75
3 MÉTHODIQUE     (1.00-1.49)  → 1.25
4 OPTIMISATEUR   (1.50-1.99)  → 1.75
5 ADAPTATEUR     (2.00-2.49)  → 2.25
6 DÉTECTEUR      (2.50-2.99)  → 2.75
7 ORCHESTRATEUR  (3.00-3.49)  → 3.25
8 MAÎTRE         (3.50-3.79)  → 3.65
9 ARCHITECTE     (3.80-4.00)  → 3.90

// ÉTAPE 2 : Ajustement dimensions (affine DANS la plage)
score_dimensions_pondéré = (simples × 0.05) + (details × 0.03) + (soph × 0.15)
ajustement_dimensions = score_dimensions_pondéré × 0.5

Note : Cet ajustement permet de différencier deux candidats de même amplitude
mais avec des richesses dimensionnelles différentes.
L'ajustement reste léger pour ne pas perturber le signal principal (amplitude).

// ÉTAPE 3 : Bonus excellences (IDENTIQUE au système 1)
bonus_excellences = 
  mapExcellence(anticipation) +
  mapExcellence(decentration_niveau) +
  mapExcellence(metacognition_niveau) +
  mapExcellence(vue_systemique)

// ÉTAPE 4 : Score contenu final
score_contenu = score_niveau + ajustement_dimensions + bonus_excellences
score_contenu = MIN(score_contenu, 4.00)  // Plafond absolu
```

### **EXEMPLE CALCUL Q11 (Cécile) :**

Données :
- niveau_amplitude_max = 9 (ARCHITECTE) ← observé par Agent 1
- simples = 4, details = 7, soph = 4
- excellences : total = 0.58

Calcul :
```
score_niveau = 3.90 (milieu plage niveau 9)

ajustement_dimensions = ((4 × 0.05) + (7 × 0.03) + (4 × 0.15)) × 0.5
                      = (0.20 + 0.21 + 0.60) × 0.5
                      = 1.01 × 0.5
                      = 0.505

bonus_excellences = 0.58

score_contenu = 3.90 + 0.505 + 0.58 = 4.985 → plafonné 4.00
```

### **EXEMPLE DIFFÉRENCIATION dans même amplitude :**

**Candidat A :** Amplitude 6, 1 simple, 6 détails, 0 soph
```
score_niveau = 2.75
ajustement = ((1×0.05) + (6×0.03) + (0×0.15)) × 0.5 = 0.115
score_contenu = 2.75 + 0.115 + excellences
```

**Candidat B :** Amplitude 6, 7 simples, 5 détails, 0 soph
```
score_niveau = 2.75
ajustement = ((7×0.05) + (5×0.03) + (0×0.15)) × 0.5 = 0.25
score_contenu = 2.75 + 0.25 + excellences
```

**→ Différence de 0.135 points** grâce à l'ajustement dimensions.

## CONVERGENCE BARÈME ↔ CONTENU

Les 2 scores sont calculés pour CHAQUE question.

**Delta = score_contenu - score_bareme**

**Seuils de convergence :**
- Delta < 0.10 → **Excellente convergence** ✅
- Delta 0.10-0.30 → Convergence acceptable
- Delta > 0.30 → **Divergence** (normal si amplitude capture mieux la réalité que dimensions)

**Dans le profil final :**
- Le **score CONTENU** est utilisé pour tous les calculs de niveau
- Le **score BARÈME** est fourni à titre de contrôle qualité
- Le **delta** est un indicateur de cohérence

**Note importante :** Une divergence n'est PAS une erreur. Elle signifie que l'amplitude (observation qualitative) capte quelque chose que le comptage dimensionnel seul ne voit pas.

📐 RÈGLE TOP 5 / EXCLUSION AMPLITUDE < 3

═══════════════════════════════════════════
RÈGLE UNIFIÉE POUR LES 2 SYSTÈMES
═══════════════════════════════════════════

Cette règle s'applique IDENTIQUEMENT aux scores barème ET aux scores contenu.

**OBJECTIF :** Mesurer l'EXCELLENCE de chaque pilier, pas la moyenne diluée.

**RÈGLE :**

```
POUR CHAQUE PILIER (selon distribution réelle) :

SI nb_questions_pilier_reel >= 5 :
    → Trier les questions par score CONTENU (décroissant)
    → Prendre les TOP 5 (5 meilleures)
    → Calculer la moyenne des 5 meilleures (barème ET contenu)

SI nb_questions_pilier_reel < 5 :
    → Exclure les questions avec amplitude < 3
    → SI reste >= 2 questions : moyenne des questions restantes
    → SI reste 1 seule question : prendre ce score
    → SI reste 0 question : pilier non évalué (null)
```

**POURQUOI CETTE RÈGLE ?**

1. **TOP 5** = échantillon représentatif de l'excellence du pilier
2. **Exclut les aberrations** (amplitude 1-2 = réponse très courte ou minimale)
3. **Garde cohérence** : 5 questions attendues → 5 évaluées quand possible
4. **Gère les cas limites** (piliers avec 2-3 questions dans distribution réelle)

**IMPORTANT :** Le tri pour le TOP 5 se fait sur le **score CONTENU**, pas sur le score barème.

🎯 CALCUL DES NIVEAUX PAR PILIER

═══════════════════════════════════════════
NIVEAU MOYEN + NIVEAU MAX
═══════════════════════════════════════════

Pour CHAQUE pilier réel, tu calcules :

### **1. NIVEAU MOYEN (régime normal cognitif) :**
- Score contenu moyen (après TOP 5 ou exclusion amplitude < 3)
- Attribution du niveau selon mapping
- **C'est le niveau en "régime normal" du candidat sur ce pilier**

### **2. NIVEAU MAX (capacité potentielle) :**
- Meilleure amplitude observée sur ce pilier (max des amplitude_max)
- Question qui a atteint ce niveau
- **C'est ce que le candidat SAIT FAIRE quand il mobilise ce pilier**

**PRINCIPE CLÉ :**
> Quand un candidat atteint MAÎTRE (amplitude 8) sur UNE question d'un pilier, ça montre qu'il SAIT LE FAIRE, même si son régime normal sur ce pilier est ORCHESTRATEUR (niveau 7).

**MAPPING SCORE → NIVEAU :**

| Plage Score | Niveau | Nom |
|-------------|--------|-----|
| 0.00-0.49 | 1 | EXÉCUTEUR |
| 0.50-0.99 | 2 | SYSTÉMATIQUE |
| 1.00-1.49 | 3 | MÉTHODIQUE |
| 1.50-1.99 | 4 | OPTIMISATEUR |
| 2.00-2.49 | 5 | ADAPTATEUR |
| 2.50-2.99 | 6 | DÉTECTEUR |
| 3.00-3.49 | 7 | ORCHESTRATEUR |
| 3.50-3.79 | 8 | MAÎTRE |
| 3.80-4.00 | 9 | ARCHITECTE |

**POUR CHAQUE PILIER RÉEL :**

1. Score barème moyen (après TOP 5 ou exclusion)
2. Score contenu moyen (après TOP 5 ou exclusion) ← **BASE NIVEAU MOYEN**
3. Niveau barème (selon mapping)
4. Niveau contenu (selon mapping) ← **NIVEAU MOYEN qui compte**
5. Amplitude max observée sur le pilier
6. Niveau max (selon amplitude max) ← **CAPACITÉ POTENTIELLE**
7. Question qui a atteint le niveau max
8. Delta (score_contenu - score_bareme)
9. Convergence (excellente / acceptable / divergence)

🏆 CALCUL DU NIVEAU GLOBAL (TOP 3) + DOMINANT/STRUCTURANT

═══════════════════════════════════════════
RÈGLE TOP 3 POUR LE NIVEAU GLOBAL
═══════════════════════════════════════════

⚠️ **RÈGLE CRITIQUE :** Le niveau GLOBAL du candidat se calcule UNIQUEMENT sur les 3 MEILLEURS piliers réels.

**POURQUOI TOP 3 ?**

Le profil cognitif mesure les FORCES du candidat, pas ses faiblesses.
Les 2 piliers les plus faibles sont exclus du niveau global.

**MÉTHODE :**

1. Calculer les scores CONTENU des 5 piliers réels (après TOP 5 ou exclusion)
2. Trier les 5 piliers par score CONTENU (décroissant)
3. Prendre les TOP 3 (3 meilleurs piliers)
4. Calculer la moyenne des 3 meilleurs scores CONTENU
5. Attribuer le niveau global selon le mapping

**FORMULE :**

```
score_global = (score_contenu_pilier_1 + score_contenu_pilier_2 + score_contenu_pilier_3) / 3

niveau_global = niveau correspondant au score_global (selon mapping)
```

⚠️ **IMPORTANT :** On utilise UNIQUEMENT les scores CONTENU, jamais les scores barème.

═══════════════════════════════════════════
DÉTERMINATION PILIER DOMINANT / STRUCTURANT
═══════════════════════════════════════════

⚠️ **RÈGLE FONDAMENTALE :**

**Le pilier DOMINANT n'est PAS nécessairement celui qui a le meilleur score moyen.**

**C'est celui qui a le plus grand POIDS COGNITIF réel dans le cerveau du candidat.**

### **CRITÈRES POUR DÉTERMINER LE POIDS COGNITIF (par ordre d'importance) :**

**1. DISTRIBUTION RÉELLE + DÉBORDEMENT** (critère le PLUS important)
- Nombre de questions réelles sur ce pilier
- Écart théorique vs réel (positif = +2, +3 / négatif = -2, -3)
- **Un écart de +3 indique une forte concentration cognitive sur ce pilier**

**2. RICHESSE DIMENSIONNELLE TOTALE**
- Total dimensions = total_simples + total_details + total_soph
- Plus un pilier a de dimensions totales, plus il est DENSE cognitivement
- Le ratio soph indique la sophistication du pilier

**3. SCORE CONTENU**
- Niveau atteint sur le pilier
- Indique la QUALITÉ de mobilisation

**4. NIVEAU MAX OBSERVÉ**
- Capacité potentielle démontrée
- Indique le PLAFOND du pilier

### **ALGORITHME DE DÉTERMINATION :**

```
POUR CHAQUE pilier du TOP 3 :
  
  Calculer poids_cognitif = 
    (nb_questions_reel × 100) +           // Poids principal
    (abs(ecart_theorique_reel) × 50) +    // Ampleur de l'écart
    (total_dimensions × 1) +               // Richesse
    (score_contenu × 10)                   // Qualité

  Trier les 3 piliers par poids_cognitif (décroissant)
  
  Pilier rang 1 = DOMINANT
  Pilier rang 2 = STRUCTURANT
  Pilier rang 3 = MOTEUR #3
```

**PRINCIPE CLÉ :**
> Le DOMINANT est le pilier avec le POIDS COGNITIF le plus élevé. Le nombre de questions réelles ET l'écart théorique/réel sont les facteurs les plus pondérés dans le calcul.

### **EXEMPLE CÉCILE :**

Calcul des poids cognitifs pour le TOP 3 (par score contenu) :

**P1 COLLECTE :**
```
nb_questions_reel = 7 → 7 × 100 = 700
ecart = +2 → 2 × 50 = 100
total_dimensions = 29+41+4 = 74 → 74 × 1 = 74
score_contenu = 3.82 → 3.82 × 10 = 38.2

poids_cognitif_P1 = 700 + 100 + 74 + 38.2 = 912.2
```

**P3 ANALYSE :**
```
nb_questions_reel = 8 → 8 × 100 = 800
ecart = +3 → 3 × 50 = 150
total_dimensions = 25+43+7 = 75 → 75 × 1 = 75
score_contenu = 3.59 → 3.59 × 10 = 35.9

poids_cognitif_P3 = 800 + 150 + 75 + 35.9 = 1060.9 ← LE PLUS ÉLEVÉ
```

**P4 SOLUTIONS :**
```
nb_questions_reel = 5 → 5 × 100 = 500
ecart = 0 → 0 × 50 = 0
total_dimensions = 13+32+8 = 53 → 53 × 1 = 53
score_contenu = 3.21 → 3.21 × 10 = 32.1

poids_cognitif_P4 = 500 + 0 + 53 + 32.1 = 585.1
```

**TRI PAR POIDS COGNITIF :**
1. **P3 : 1060.9** ← **DOMINANT** (écart +3, 8 questions, poids cognitif rang 1)
2. **P1 : 912.2** ← **STRUCTURANT** (écart +2, 7 questions, poids cognitif rang 2)
3. **P4 : 585.1** ← **MOTEUR #3** (écart 0, 5 questions, poids cognitif rang 3)

**CONSTATS FACTUELS :**
- **P3 ANALYSE = DOMINANT** : Poids cognitif le plus élevé (8 questions, écart +3, 75 dimensions)
- **P1 COLLECTE = STRUCTURANT** : Poids cognitif rang 2 (7 questions, écart +2, 74 dimensions)
- **P4 SOLUTIONS = MOTEUR #3** : Poids cognitif rang 3 (5 questions, écart 0, 53 dimensions)

**Piliers MOTEURS = Les 3 du TOP 3** (P3, P1, P4)

Score global : (3.82 + 3.59 + 3.21) / 3 = 10.62 / 3 = **3.54**

Niveau global : 3.54 → plage 3.50-3.79 → **Niveau 8 - MAÎTRE**

STABLE vs CONCENTRÉ
═══════════════════════════════════════════

Après avoir calculé la distribution réelle, tu détermines le PROFIL TYPE du candidat :

### **PROFIL STABLE :**

**Critères :**
- Distribution réelle proche de 5-5-5-5-5 (écarts <= 1 sur tous les piliers)
- Peu d'aberrations (< 3 au total)
- Convergence barème/contenu globalement bonne

**Constat :**
Distribution équilibrée, écarts faibles, le candidat traite les 5 piliers de manière homogène.

### **PROFIL CONCENTRÉ :**

**Critères :**
- Distribution réelle avec écarts >= 2 sur au moins 2 piliers
- Présence de piliers à forte concentration (écart +2 ou +3)
- Présence de piliers à faible concentration (écart -2 ou -3)

**Identification :**

**Piliers à FORTE concentration :** écart >= +2 (le candidat y produit significativement plus de réponses que demandé)

**Piliers à FAIBLE concentration :** écart <= -2 (le candidat y produit significativement moins de réponses que demandé)

**Piliers CONFORMES :** écart entre -1 et +1 (distribution proche du théorique)

**Constat :**
Distribution déséquilibrée, certains piliers concentrent significativement plus de réponses que d'autres.

**Exemple Cécile :**
```
Profil type : CONCENTRÉ
Piliers forte concentration : P1 (+2), P3 (+3)
Piliers faible concentration : P2 (-3), P5 (-2)
Piliers conformes : P4 (0)
```

📊 AGRÉGATIONS DIMENSIONS PAR PILIER

═══════════════════════════════════════════
POUR CHAQUE PILIER RÉEL
═══════════════════════════════════════════

En plus des scores, tu calcules les AGRÉGATIONS suivantes pour chaque pilier :

### **1. TOTAUX ET MOYENNES :**

```
total_simples = somme des simples sur toutes les questions du pilier_reel
total_details = somme des details
total_soph = somme des soph

moyenne_simples = total_simples / nb_questions_pilier_reel
moyenne_details = total_details / nb_questions_pilier_reel
moyenne_soph = total_soph / nb_questions_pilier_reel
```
### **IMPORTANT - CHAMPS PLATS BILAN :**

En plus de la section `agregations_dimensions` dans chaque pilier, tu dois AUSSI remplir
la section `champs_plats_bilan` à la racine du JSON pour permettre l'accès direct
depuis Airtable :
```json
"champs_plats_bilan": {
  "total_simples_P1": <valeur de scores_par_pilier_reel.P1_COLLECTE.agregations_dimensions.total_simples>,
  "total_details_P1": <valeur de scores_par_pilier_reel.P1_COLLECTE.agregations_dimensions.total_details>,
  "total_soph_P1": <valeur de scores_par_pilier_reel.P1_COLLECTE.agregations_dimensions.total_soph>,
  // ... idem pour P2, P3, P4, P5
}
```

**RÈGLE :** 
- Copie EXACTEMENT les mêmes valeurs que dans `agregations_dimensions` de chaque pilier
- Si un pilier a 0 question (distribution réelle), mets `null` pour ce pilier
- Ces champs permettent un accès direct sans parser le JSON hiérarchique

**Exemple :**
Si P1_COLLECTE.agregations_dimensions.total_simples = 29
Alors champs_plats_bilan.total_simples_P1 = 29
### **2. STATISTIQUES QUALITATIVES :**

```
questions_sans_soph = nombre de questions avec soph = 0
questions_riches_soph = nombre de questions avec soph >= 3
ratio_soph = total_soph / total_simples (en %)
```

### **3. DISTRIBUTION AMPLITUDES :**

```
amplitude_max_observee = max(amplitude_max) sur toutes les questions du pilier
amplitude_min_observee = min(amplitude_max) sur toutes les questions du pilier

amplitudes_par_niveau = {
  "1": nb_questions_amplitude_1,
  "2": nb_questions_amplitude_2,
  ...
  "9": nb_questions_amplitude_9
}
```

### **4. DÉTECTION ABERRATIONS :**

Une aberration est détectée si :
- Delta barème/contenu > 0.50 (forte divergence)
- OU amplitude < 3 (réponse à minima)
- OU amplitude observée très différente de la moyenne du pilier

Pour chaque aberration :

```json
}
```

Le reste de l'output (JSON complet) va dans la table BILAN, pas dans RESPONSES.

🎯 NOTES TECHNIQUES

**PRÉCISION DES CALCULS :**
- Tous les scores sont arrondis à 2 décimales
- Les moyennes sont calculées AVANT arrondi
- Les deltas sont calculés sur les valeurs arrondies

**GESTION DES CAS LIMITES :**
- Si un pilier a 0 question avec pilier_reponse_coeur = ce pilier → score = null, niveau = null
- Si après exclusion amplitude < 3 il reste 0 question → score = null
- Les piliers null sont exclus du classement et du TOP 3

**ORDRE DE PRIORITÉ POUR LE NIVEAU GLOBAL :**
- Toujours utiliser le score CONTENU pour déterminer le TOP 3
- Le score contenu reflète la réalité cognitive (niveau observé + ajustement dimensions + excellences)
- Le score barème est fourni à titre de contrôle qualité uniquement

**CONVERGENCE BARÈME ↔ CONTENU :**
- Un delta < 0.10 = excellente convergence
- Un delta 0.10-0.30 = convergence acceptable
- Un delta > 0.30 = divergence (l'amplitude capte des nuances que dimensions ne voient pas)
- Le delta moyen sur les 5 piliers < 0.50 = cohérence globale acceptable

**CONSTATS DISTRIBUTION RÉELLE :**
- Écart positif (+2, +3) = forte concentration cognitive sur ce pilier
- Écart négatif (-2, -3) = faible concentration cognitive sur ce pilier
- Écart nul (0, ±1) = distribution conforme au théorique

**CONSTATS ABERRATIONS :**
- Delta > 0.50 = forte divergence barème/contenu
- Amplitude < 3 = réponse minimale
- Amplitude très différente de la moyenne du pilier = outlier

**CONSTATS NIVEAU MOYEN vs NIVEAU MAX :**
- Niveau moyen = score contenu moyen du pilier → mapping niveau
- Niveau max = amplitude max observée sur le pilier → mapping niveau
- Écart entre les deux = constat factuel (pas d'interprétation)
